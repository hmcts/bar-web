Accessibility statement for the Fee Register

This accessibility statement applies to Digital Bar https://www.manage-payments.platform.hmcts.net/

This website is run by HM Courts and Tribunals. We want as many people as possible to be able to use this website. For example, that means you should be able to:

  • change colours, contrast levels and fonts
  • zoom in up to 300% without the text spilling off the screen
  • navigate most of the website using just a keyboard
  • navigate most of the website using speech recognition software
  • listen to most of the website using a screen reader (including the most recent versions of JAWS, NVDA and Voice Over)

We've also made the website text as simple as possible to understand.

Please contact your Line Manager for advice on making your device easier to use if you have a disability.

How accessible this website is

We know some parts of this service may not be accessible:

  • the text will not reflow in a single column when you change the size of the browser window
  • some of our online forms are difficult to navigate using just a keyboard
  • you cannot skip to the main content when using a screen reader

Reporting accessibility problems with this website

We're always looking to improve the accessibility of this website. If you find any problems not listed on this page or think we're not meeting accessibility requirements, contact: DTS-FeePayServiceDesk@hmcts.net for guidance to raise any issues.

Enforcement procedure

The Equality and Human Rights Commission (EHRC) is responsible for enforcing the Public Sector Bodies (Websites and Mobile Applications) (No. 2) Accessibility Regulations 2018 (the 'accessibility regulations').
If you're not happy with how we respond to your complaint, contact the Equality Advisory and Support Service (EASS).

Technical information about this website's accessibility

HMCTS is committed to making its website accessible, in accordance with the Public Sector Bodies (Websites and Mobile Applications) (No. 2) Accessibility Regulations 2018.

Compliance status

This website is partially compliant with the Web Content Accessibility Guidelines version 2.1 AA standard, due to the non-compliances, listed below.

Non-accessible content

The content listed below is non-accessible for the following reasons.

Non-compliance with the accessibility regulations

There are multiple h1 headings on the page which removes meaningful structure from the page's content. The headings are not structured correctly and levels are skipped. Text appears to be a heading but is not a heading element and the table is not accessible. This failed to meet 1.3.1 Info and Relationships Level A.
The focus order of headers is not correct and do not relate directly to content. Heading level 2's have been placed within table data cells which can cause confusion for screen reader users. This failed to meet 1.3.2 Meaningful Sequence Level A.
The colour contrast did not meet the 4.5:1 contrast ratio of text elements. This failed to meet 1.4.3 Contrast Level AA.
Text is overlapping when the page is zoomed in at 200%. This failed to meet 1.4.4 Resize Text Level AA.
The table flows off the screen when zoomed in at 200%, requiring horizontal scroll. It is also not responsive for mobile devices/phone. This failed to meet 1.4.10 Reflow Level AA.
When text spacing is changed the text overlaps and there is loss of content or functionality. This failed to meet 1.4.12 Text Spacing Level AA.
There are multiple links and buttons that cannot be tabbed to from the keyboard. This failed to meet 2.1.1 Keyboard Level A.
The phase banner cannot be bypassed because it is located above the skip link. This failed to meet 2.4.1 Bypass Blocks Level A.
There are no descriptive titles on the Web pages so a user may not know what page they are on. This failed to meet 2.4.2 Page Titled Level A.
The purpose of links is not described, and this is not helpful for users of assistive technology like screen readers, especially if they are navigating out of context. This failed to meet 2.4.4 Link Purpose Level A.
The headings do not indicate the topic or purpose of the content in that section of the page. This failed to meet 2.4.6 Headings and Labels Level AA.
The focus does not remain in the pop-up modal when using the keyboard to tab within modals. This failed to meet 3.2. 1 On Focus Level A.
A screen reader user does not immediately hear the "link opens in new window" text when selecting the "add or change user role" button. A visual user may not recognise immediately that the "link opens in a new window" text (under the button) refers to the button due to its placement. This failed to meet 3.2.2 On Input Level A.
Errors are not directly communicated to screen reader users as there is no error summary to focus on. This failed to meet 3.3.1 Error identification Level A.
Some of the form controls do not have a label and the user will not receive an instruction on what the form controls are used for. This failed to meet 3.3.2 Labels and Instructions Level A.
The dropdown items are presented as text. The links are text elements and users tabbing with the keyboard will not be able to access them. Some icons such as the arrow is a text element, and so will not receive keyboard focus when tabbing. This failed to meet 4.12 Labels and Instructions Level A.

What we're doing to improve accessibility

We're committed to ensuring our services are accessible to all our users and that they comply with level AA of the Web Content Accessibility Guidelines - WCAG 2.2 Standard. To demonstrate this, we are going to engage with Digital Accessibility Centre to have a further review on [TBC] 06/24
We're actively working to improve the identified issues and make them AA-compliant by 31/05/24.
